# Auto-Generate JSON Schema Scripts

This directory contains utility scripts to transform the Wi_Wallet Design System documentation (Markdown) into a machine-readable JSON Schema. This schema is primarily used by the MCP Server to provide context to AI agents.

## 📂 Files Overview

*   **`parser.js`**: The core logic that reads raw Markdown text and extracts structured data (Widgets, Props, Design Tokens) using Regular Expressions.
*   **`generate-schema.js`**: The main execution script. It reads the source files, calls the parser, and writes the output JSON.
*   **`check-v3-boundaries.js`**: Enforces Theme/Widget/Skill V3 isolation rules and supports base-ref-aware changed-file checks for CI.
*   **`check-v3-boundaries.test.js`**: Covers allowed and rejected V3 boundary scenarios with Node's built-in test runner.

## 🚀 Usage

You typically run these scripts via `npm` from the **project root**:

```bash
# Generate schema once
npm run generate-schema

# Watch for changes and auto-regenerate
npm run generate-schema:watch

# Check Theme/Widget/Skill V3 boundaries
npm run check:v3-boundaries
npm run test:v3-boundaries

# Compare changed paths with a review base
npm run check:v3-boundaries -- --base-ref origin/main
```

### Direct Execution
If needed, you can run them directly with node:
```bash
node scripts/generate-schema.js
```

## ⚙️ Configuration

The input/output paths are defined in `generate-schema.js`:

```javascript
const config = {
  inputFiles: {
    context: "./CODEBASE_CONTEXT.md",
    widgets: "./WIDGETS_GUIDE.md"
  },
  outputDir: "./docs",
  outputFile: "schema.json"
};
```

## 🛠️ How to Customize

### Adding new Widget Documentation format
If you change how you document widgets in `WIDGETS_GUIDE.md`, you need to update the regex in `parser.js`.

The current parser implementation looks for:
1.  **Level 2 Headers** (`## WidgetName`) as the start of a widget.
2.  **Ignored Sections** (e.g., "Table of Contents") are skipped.
3.  **Properties Table** (`| Property | Type | ...`) to extract API props.
4.  **Usage Example** (` ```dart ... ``` `) matching the `Usage` subsection.

**Example Modification:**
If you want to extract a "Category" explicitly defined in markdown like `Category: Input`, you would add a regex match in `parseMarkdownWidgets` function inside `parser.js`.

### Adding new Design Tokens
To support new token categories (e.g., "Elevations"), update `extractDesignTokens` in `parser.js`. Currently, it scans `CODEBASE_CONTEXT.md` for specific "Color Scheme" sections.

## 📦 Output
The script generates `docs/schema.json`. **Do not edit this file manually**, as it will be overwritten the next time the script runs.
